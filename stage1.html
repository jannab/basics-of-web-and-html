Lesson 1: The Basics of the Web and HTML

How the Web Works

The web consists of a lot of computers and servers all over the world. When someone tries to reach a web page like udacity.com, the computer actually sends a HTTP Request to a server. The server looks up the right HTML document and sends it back to the computer. Then, the browser of the person interprets the HTML code and shows the web page accordingly.

I didn't learn any new concepts here, but this was a great quick review, which was definitely not boring.

HTML

HTML is an abbreviation for Hypertext Markup Language. Even though the web contains videos, pictures and much more, most of it is written in HTML and videos and pictures are embedded in the HTML code. In HTML we have text, that describes the texts you see and the markup describes the way it looks.

This was a great overview. It helped me refresh the basic HTML knowledge I have.

Tags and Elements

Elements in HTML are defined using tags. An HTML document consits of several elements. A title for example is an element and we would use a h1 tag for that.

Why Computers are Stupid

Computers just do what you tell them to do, that makes them kind of stupid. Therefore the programmer has to be careful with it's code and small mistakes can cause big problems.

Inline vs Block Elements

Block elements have an "invisible box" around them. Therefore, block elements create new paragraphs. Inline elements can be in a paragraph without having to start a new one.

Lesson 2: Creating a Structured Document with HTML

Developer Tools (in the Browser)

It is possible to see the source code of every web page through the developer tools of the browser.

The "tree-like structure" of HTML

HTML elements can have one or more elements in them. One could say the inner elements of one element are their children. Then it is easy to imagine the elements as persons in a family tree. Therefore HTML has a tree-like structure.

In a browser, HTML is made of boxes, every shape that you see is in some box. Boxes can be nested in other boxes through the HTML code.

The relationships between indented HTML and boxes

When you see HTML code as a text, the code has different levels of indentation. This is due to the fact, that the deeper the boxes represented by the elements are, the more intended they are.

Text Editors (for programming)

Text editors for programming highlight some things in different colors, to make the code more readable. Also some help with automatically adding closing tags or insert a whole tag when you type in a few letters.

One example is Sublime Text (you can download a free version here).
